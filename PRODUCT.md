# Product Contract

## User

A project professional has built an automation, dashboard or AI-assisted workflow and wants another person or a team to rely on it. An IT, Data, Security or delivery reviewer needs an evidenced pre-filter before deciding what review is still required.

## Journey

1. State intended reliance: personal, team or production.
2. Add a local folder or load the safe sample.
3. Inspect the fix-first list, data flow, coverage and exact evidence.
4. Correct the build outside Newco and scan again.
5. A named person records the permitted reliance and conditions.
6. Export an assurance record tied to a source fingerprint.

## Release gates

- No source upload or execution in the browser prototype.
- No severe finding without exact local evidence.
- No more than three fix-first items.
- Missing coverage remains visible.
- Human approval cannot be inferred.
- Production approval is unavailable in the public prototype.
- Deterministic tests cover rule behavior and reliance outcomes.
- Desktop, 390px and no-JavaScript browser paths are checked.

## Commercial hypothesis

A controlled pilot scans one sanitised workflow, includes a context session and human-reviewed readout, then re-scans after remediation. Proposed test price: £2,500-£5,000. Do not turn this into published list pricing until willingness to pay is validated.
