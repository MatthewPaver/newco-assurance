# Eight-minute facilitated demonstration

## Set-up

Use the weekly status automation first. It gives the builder and reviewer a familiar task and produces a clear Hold-to-Ready journey.

## 0:00–1:00 · Put the buyer in the situation

“A project controls analyst built this workflow with an AI coding tool. It reads the weekly CSV, drafts a status pack and works in the demonstration. The programme now wants twelve people to use it.”

Ask: “What would you need to see before colleagues rely on it?”

## 1:00–2:00 · State intended reliance

Choose **Team**. Leave the owner blank because that reflects the submitted fixture. Load **Weekly status automation**.

Explain that the public demonstration reads the embedded fixture in the browser. It uploads and executes nothing.

## 2:00–4:00 · Review three findings

Open:

1. the redacted credential finding;
2. the external destination and visible data flow;
3. the missing ownership or failure evidence.

Show the exact file, line, provenance, action and coverage limits.

Do not defend the numeric indicator. It is an uncalibrated static-rule indicator. The result and evidence matter.

## 4:00–5:30 · Reassess

Select **Load documented fixes and reassess**.

Explain that the button loads a second share-safe fixture. It does not edit a user's source. Compare:

- original and current source fingerprints;
- result before and after;
- resolved finding identifiers;
- remaining limitations.

## 5:30–6:30 · Record authority

Enter an accountable role, choose a permitted reliance decision and state conditions. Download the assurance record.

Point out that the public product cannot record production approval.

## 6:30–7:15 · Test the value

Open the buyer calculator. Replace the defaults with the buyer's own workflow volume, review hours, rework and loaded cost.

Keep approval delay visible but unpriced unless the buyer supplies a daily value.

## 7:15–8:00 · Ask for evidence

Ask:

1. Which evidence would you trust?
2. Which finding would your existing process already catch?
3. What would IT, Data or Risk still need?
4. Did the report change the permitted use?
5. Would you test this on three sanitised workflows for £7,500 + VAT?

Record the answers verbatim. A polite discussion does not count as demand evidence.
